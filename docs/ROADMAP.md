# Roadmap

This roadmap records direction, not fixed delivery dates. Priorities should be confirmed in issues before implementation.

## Foundation

- Maintain reliable lint, type-check, and production-build gates.
- Establish consistent issue, pull request, review, and release practices.
- Add an automated test strategy for shared content and critical user journeys.

## Content and localization

- Complete editorial review across English, Japanese, and Simplified Chinese.
- Define a repeatable process for adding and validating localized content.
- Replace placeholder and coming-soon material as releases are approved.

## Experience and accessibility

- Audit keyboard navigation, focus states, semantics, contrast, and reduced motion.
- Validate responsive behavior and performance on representative devices.
- Improve metadata, social previews, and discoverability without changing brand direction.

## Operations

- Document preview, production deployment, rollback, and incident ownership.
- Add dependency and security maintenance policies.
- Introduce monitoring and measurable performance budgets.

## Decision rule

Each roadmap item should become a scoped issue with an owner, acceptance criteria, constraints, and validation steps before work begins.
